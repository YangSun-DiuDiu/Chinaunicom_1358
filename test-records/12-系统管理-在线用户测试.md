# 12-系统管理-在线用户测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 在线用户管理 |
| 涉及页面 | `views/monitor/online/index.vue`（前端有定义，需确认路由） |
| API 接口 | 2个（list、forceLogout） |
| 权限要求 | `monitor:online:list`、`monitor:online:forceLogout` |

### API 清单

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/monitor/online/list` | `monitor:online:list` | 在线用户列表 |
| DELETE | `/monitor/online/{tokenId}` | `monitor:online:forceLogout` | 强制下线 |

## 2. 前置条件

- 以 admin 账号登录
- 至少同时有2个用户在线

## 3. 测试用例

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| ONLINE-001 | 在线用户列表加载 | 进入在线用户页面 | 显示当前在线的用户列表 |
| ONLINE-002 | 列表字段完整 | 查看列表列 | 包含会话编号/用户名称/部门/登录IP/地址/浏览器/操作系统/登录时间 |
| ONLINE-003 | 强制下线-其他用户 | 1.对某个非本用户的在线记录点击强退 2.确认 | 该用户被强制下线，需要重新登录 |
| ONLINE-004 | 列表实时更新 | 新用户登录/登出后刷新 | 列表反映最新在线状态 |
| ONLINE-005 | 搜索筛选 | 输入用户名称搜索 | 返回匹配的在线用户 |

## 4. 边界与异常测试

| 用例ID | 测试名称 | 条件/步骤 | 预期结果 |
|--------|----------|-----------|----------|
| ONLINE-B01 | 强退自己 | 尝试强退当前登录用户 | 提示"不能强退自己"或当前用户被踢出 |
| ONLINE-B02 | 强退已离线的用户 | 用户刚刚离线后尝试强退 | 提示用户已离线 |
| ONLINE-E01 | 无在线用户 | 所有用户登出后查看 | 显示空列表 |

## 5. 权限测试

| 用例ID | 条件 | 预期结果 |
|--------|------|----------|
| ONLINE-P01 | 无`monitor:online:list` | 页面不可访问 |
| ONLINE-P02 | 无`monitor:online:forceLogout` | 强退按钮不显示 |
