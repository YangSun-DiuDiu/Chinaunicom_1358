# 06-系统管理-部门管理测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 部门管理 |
| 涉及页面 | `views/system/dept/index.vue` |
| API 接口 | 6个（list、listDeptExcludeChild、getDept、addDept、updateDept、updateDeptSort、delDept） |
| 权限要求 | `system:dept:list`、`system:dept:add`、`system:dept:edit`、`system:dept:remove` |

### API 清单

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/system/dept/list` | `system:dept:list` | 部门列表（树形） |
| GET | `/system/dept/list/exclude/{deptId}` | `system:dept:list` | 排除子节点的部门列表 |
| GET | `/system/dept/{deptId}` | `system:dept:query` | 部门详情 |
| POST | `/system/dept` | `system:dept:add` | 新增部门 |
| PUT | `/system/dept` | `system:dept:edit` | 修改部门 |
| PUT | `/system/dept/updateSort` | `system:dept:edit` | 保存排序 |
| DELETE | `/system/dept/{deptId}` | `system:dept:remove` | 删除部门 |

## 2. 前置条件

- 以 admin 账号登录
- 存在至少2级部门层级

## 3. 页面元素检查

| 元素 | 类型 | 检查项 |
|------|------|------|
| 搜索栏-部门名称 | el-input | 可输入搜索 |
| 搜索栏-状态 | el-select | 正常/停用 |
| 新增按钮 | el-button | `v-hasPermi="['system:dept:add']"` |

| 树形表格 | el-table (tree) | 树形结构显示，层级缩进 |
| 表格列-部门名称 | el-table-column | 显示 |
| 表格列-排序 | el-table-column | 显示 |
| 表格列-状态 | el-table-column | 显示 |
| 表格列-创建时间 | el-table-column | 显示 |
| 表格列-操作 | el-table-column | 新增子部门/编辑/删除（每行都有新增按钮） |

## 5. 边界与异常测试

| 用例ID | 测试名称 | 步骤/条件 | 预期结果 |
|--------|----------|-----------|----------|
| DEPT-B01 | 名称必填 | 不填部门名点击确定 | 提示必填 |
| DEPT-B02 | 上级部门不能选自己 | 编辑时上级部门选择自己 | 不可选或提交后提示错误 |
| DEPT-B03 | 删除有子部门的部门 | 尝试删除非叶子部门 | 提示"存在下级部门，不允许删除" |
| DEPT-B04 | 删除有用户的部门 | 尝试删除挂载了用户的部门 | 提示"存在用户，不允许删除" |
| DEPT-B05 | 排序值修改 | 修改某部门的排序值 | 列表排序刷新正确 |
| DEPT-B06 | 部门名称重复 | 同一父级下创建同名部门 | 提示"部门名称已存在" |
| DEPT-X01 | 删除不存在的部门 | 删除已被他人删除的部门 | 提示"部门不存在" |
