# 09-系统管理-参数配置测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 参数配置 |
| 涉及页面 | `views/system/config/index.vue`（前端有API定义，需确认路由配置） |
| API 接口 | 7个（list、getConfig、getConfigKey、addConfig、updateConfig、delConfig、refreshCache） |
| 权限要求 | `system:config:list`、`system:config:add`、`system:config:edit`、`system:config:remove` |

### API 清单

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/system/config/list` | `system:config:list` | 参数列表 |
| GET | `/system/config/{configId}` | `system:config:query` | 参数详情 |
| GET | `/system/config/configKey/{configKey}` | 需登录 | 按key查值（前端通用接口） |
| POST | `/system/config` | `system:config:add` | 新增参数 |
| PUT | `/system/config` | `system:config:edit` | 修改参数 |
| DELETE | `/system/config/{configIds}` | `system:config:remove` | 删除参数 |
| DELETE | `/system/config/refreshCache` | `system:config:remove` | 刷新参数缓存 |

## 2. 前置条件

- 以 admin 账号登录
- 数据库 `sys_config` 表中有现有参数
