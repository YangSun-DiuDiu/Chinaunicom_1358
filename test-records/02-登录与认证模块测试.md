# 02-登录与认证模块测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 登录与认证 |
| 涉及页面 | `views/login.vue`（登录页）、`views/lock.vue`（锁屏页） |
| API 接口 | 7个（login、getInfo、getRouters、getCodeImg、logout、unlockScreen、getConfig） |
| 权限要求 | 无需认证（公开接口） |

### API 清单

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/login` | 用户登录 |
| GET | `/getInfo` | 获取用户信息、角色、权限 |
| GET | `/getRouters` | 获取前端路由 |
| GET | `/captchaImage` | 获取验证码 |
| POST | `/logout` | 登出 |
| POST | `/unlockscreen` | 锁屏解锁 |
| GET | `/getConfig` | 获取系统配置 |

## 2. 前置条件

- MySQL 数据库已启动并初始化
- Redis 服务已启动
- 后端服务已启动（端口 8080）
- 前端服务已启动
- 测试浏览器：Chrome / Edge

## 3. 页面元素检查

### 3.1 登录页面

| 元素 | 类型 | 检查项 |
|------|------|--------|
| 用户名输入框 | el-input | 必填、placeholder 显示、焦点样式 |
| 密码输入框 | el-input (type=password) | 必填、密码遮蔽、显示/隐藏切换 |
| 验证码输入框 | el-input | 必填 |
| 验证码图片 | img | 可点击刷新 |
| 记住我复选框 | el-checkbox | 可勾选/取消 |
| 登录按钮 | el-button | 点击触发登录 |
| 系统标题 | 文本 | 显示"智慧园区管理系统" |

### 3.2 锁屏页面

| 元素 | 类型 | 检查项 |
|------|------|--------|
| 用户头像/名称 | 显示 | 显示当前锁定用户信息 |
| 密码输入框 | el-input | 必填 |
| 解锁按钮 | el-button | 点击触发解锁 |

## 5. 边界测试用例

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| LOGIN-B01 | 用户名为空 | 1.不填用户名 2.填写密码和验证码 3.点击登录 | 提示"用户账号不能为空"或前端校验阻止 |
| LOGIN-B02 | 密码为空 | 1.填写用户名 2.不填密码 3.点击登录 | 提示"密码不能为空"或前端校验阻止 |
| LOGIN-B03 | 验证码为空 | 1.填写用户名密码 2.不填验证码 3.点击登录 | 提示"验证码不能为空" |
| LOGIN-B04 | 用户名超长 | 1.输入100个字符的超长用户名 | 正常提交，服务端返回校验失败 |
| LOGIN-B05 | 用户名含特殊字符 | 1.输入 `<script>alert(1)</script>` 用户名 | XSS过滤生效，不触发脚本 |
| LOGIN-B06 | 密码超长 | 输入500字符的超长密码 | 正常提交 |
| LOGIN-B07 | 验证码不区分大小写 | 输入大小写不一致的验证码 | 验证码校验通过（或明确失败） |
| LOGIN-B08 | 连续输错密码 | 连续5次输入错误密码 | 账户被锁定（如开启了密码重试限制） |

## 6. 异常测试用例

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| LOGIN-E01 | 不存在的用户名 | 输入不存在的用户名 | 提示"用户不存在"或"用户名或密码错误" |
| LOGIN-E02 | 停用账号登录 | 使用status=1(停用)的账号登录 | 提示"用户已停用" |
| LOGIN-E03 | 错误验证码 | 输入与图片不匹配的验证码 | 提示"验证码错误" |
| LOGIN-E04 | 过期验证码 | 等待验证码过期后再提交 | 提示"验证码已失效" |
| LOGIN-E05 | 后端服务不可用 | 关闭后端服务后点击登录 | 提示"网络错误"或"服务不可用" |
| LOGIN-E06 | 错误的JWT token | 手动修改localStorage中的token后刷新 | 跳转登录页，提示token无效 |
| LOGIN-E07 | Token过期 | 等待token过期（默认120分钟）后操作 | 自动跳转登录页 |
| LOGIN-E08 | 直接访问受保护URL | 未登录直接访问 `/system/user` | 重定向到登录页 |

## 7. 安全测试

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| LOGIN-S01 | SQL注入 | 用户名输入 `' OR '1'='1` | 登录失败，不返回任何用户数据 |
| LOGIN-S02 | 暴力破解防护 | 使用脚本连续发起登录请求 | 触发限流机制（如有的话） |
| LOGIN-S03 | 密码传输加密 | F12 Network查看登录请求 | 密码不应明文传输（前端加密或HTTPS） |
| LOGIN-S04 | XSS攻击 | 用户名输入 `<img src=x onerror=alert(1)>` | 输入被过滤或转义 |
| LOGIN-S05 | 会话固定 | 登录前后对比Cookie/Session | 登录后token应变更 |
| LOGIN-S06 | 登出后token失效 | 1.登出 2.使用旧token调用API | 返回401未授权 |
