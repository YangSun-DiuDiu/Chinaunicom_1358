# 01-测试计划总览

## 1. 测试目标

对"智慧园区管理系统"进行全面功能测试，确保所有页面、接口、按钮、导入导出、数据筛选功能均正常工作。

### 2.2 API 接口覆盖

接口类型分布：

| HTTP方法 | 数量 | 说明 |
|----------|------|------|

| DELETE | ~15 | 删除、清空、缓存刷新 |

## 3. 测试策略

### 3.1 测试层次

```
          ┌──────────────────────┐
          │   UI 交互层测试       │  ← 页面元素渲染、按钮点击、表单交互
          ├──────────────────────┤
          │   业务逻辑层测试       │  ← 数据校验、状态流转、权限控制
          ├──────────────────────┤
          │   API 接口层测试      │  ← 请求响应、错误码、数据一致性
          ├──────────────────────┤
          │   数据持久层测试       │  ← 数据库读写、事务、唯一约束
          └──────────────────────┘
```

### 3.2 测试优先级

| 优先级 | 说明 | 覆盖内容 |
|--------|------|----------|
| P0 | 核心功能 | 登录、用户CRUD、权限验证、数据安全 |
| P1 | 重要功能 | 角色/菜单/部门管理、设备管理、访客审批 |
| P2 | 一般功能 | 导入导出、日志查看、图表展示、短信配置 |
| P3 | 辅助功能 | 列显隐、主题切换、全屏、头像上传 |

### 3.3 测试数据

| 数据类型 | 要求 |
|----------|------|
| 用户数据 | 至少3个不同角色的账号 |
| 设备数据 | 至少5台设备（在线/离线/不同类型） |
| 访客数据 | 至少10条预约（各状态覆盖） |
| 日志数据 | 至少50条操作日志和登录日志 |
| 部门数据 | 至少3级部门树 |
| 菜单数据 | 至少20条菜单项 |
| 角色数据 | 至少3个角色（不同权限） |

## 6. 缺陷严重程度定义

| 等级 | 定义 | 示例 |
|------|------|------|
| P0-致命 | 系统崩溃/数据丢失/安全漏洞 | 任意用户可访问所有数据、密码明文存储 |
| P1-严重 | 核心功能无法使用 | 用户无法登录、设备无法新增 |
| P2-一般 | 功能可用但有缺陷 | 导出数据缺少字段、筛选条件无效 |
| P3-轻微 | 用户体验问题 | 提示文案错误、样式错位 |
