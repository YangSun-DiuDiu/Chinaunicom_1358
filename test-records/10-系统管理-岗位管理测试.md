# 10-系统管理-岗位管理测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 岗位管理 |
| 涉及页面 | `views/system/post/index.vue`（前端有API定义，需确认路由配置） |
| API 接口 | 5个（list、getPost、addPost、updatePost、delPost） |
| 权限要求 | `system:post:list`、`system:post:add`、`system:post:edit`、`system:post:remove` |

### API 清单

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/system/post/list` | `system:post:list` | 岗位列表 |
| GET | `/system/post/{postId}` | `system:post:query` | 岗位详情 |
| POST | `/system/post` | `system:post:add` | 新增岗位 |
| PUT | `/system/post` | `system:post:edit` | 修改岗位 |
| DELETE | `/system/post/{postIds}` | `system:post:remove` | 删除岗位 |

## 2. 前置条件

- 以 admin 账号登录
- 数据库 `sys_post` 表中有现有岗位数据

## 3. 测试用例

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| POST-001 | 岗位列表加载 | 进入岗位管理页面 | 表格加载岗位列表，分页正常 |
| POST-002 | 新增岗位 | 1.点击新增 2.填写岗位编码/名称/排序 3.保存 | 新增成功 |
| POST-003 | 编辑岗位 | 修改岗位名称后保存 | 修改成功 |
| POST-004 | 删除岗位 | 删除无用户关联的岗位 | 删除成功 |
| POST-005 | 批量删除 | 选择多个岗位后删除 | 批量删除成功 |
| POST-006 | 排序验证 | 新增多个岗位查看排序 | 列表按postSort排序 |

| 用例ID | 测试名称 | 条件/步骤 | 预期结果 |
