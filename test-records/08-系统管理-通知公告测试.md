# 08-系统管理-通知公告测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 通知公告 |
| 涉及页面 | `views/system/notice/index.vue`（前端有API定义但页面路由可能未配置） |
| API 接口 | 9个（listTop、list、getNotice、addNotice、updateNotice、delNotice、markRead、markReadAll、listNoticeReadUsers） |
| 权限要求 | `system:notice:list`、`system:notice:add`、`system:notice:edit`、`system:notice:remove` |

### API 清单

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/system/notice/listTop` | 需登录 | 首页公告（最多5条，含已读状态） |
| GET | `/system/notice/list` | `system:notice:list` | 公告列表 |
| GET | `/system/notice/{noticeId}` | `system:notice:query` | 公告详情 |
| POST | `/system/notice` | `system:notice:add` | 新增公告 |
| PUT | `/system/notice` | `system:notice:edit` | 修改公告 |
| DELETE | `/system/notice/{noticeIds}` | `system:notice:remove` | 删除公告 |
| POST | `/system/notice/markRead` | 需登录 | 标记已读 |
| POST | `/system/notice/markReadAll` | 需登录 | 批量标记已读 |
| GET | `/system/notice/readUsers/list` | 需登录 | 已读用户列表 |

## 2. 前置条件

- 以 admin 账号登录
- 确保通知公告菜单在路由中存在

## 3. 测试重点

由于通知公告页面在前端的路由配置中可能未显式配置为独立页面（主要通过首页通知栏交互），测试重点在于：

1. **API 接口测试**：验证所有通知公告相关接口的请求响应
2. **首页展示**：验证首页通知栏的公告显示和已读未读状态
3. **标记已读**：验证已读/全部已读功能

## 4. 测试用例

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| NOTICE-001 | 首页公告列表 | 1.登录后查看首页通知栏 | 显示最新公告列表（最多5条） |
