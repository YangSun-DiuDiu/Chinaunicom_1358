# 21-访客管理-审批管理测试

## 1. 测试概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 访客审批管理 |
| 涉及页面 | `views/visitor/approval/index.vue` |
| API 接口 | 1个主接口（pendingAppointment列表 + approveAppointment审批） |
| 权限要求 | `visitor:appointment:pending`（查看）、`visitor:appointment:approve`（审批） |

### API 清单

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/visitor/appointment/pending` | `visitor:appointment:pending` | 待审批列表 |
| PUT | `/visitor/appointment/approve` | `visitor:appointment:approve` | 审批操作（通过/拒绝） |

## 2. 前置条件

- 以 admin 账号登录
- 数据库中存在状态为 PENDING 的预约

## 3. 页面元素检查

| 元素 | 类型 | 检查项 |
|------|------|------|
| 待审批表格 | el-table | 仅显示状态=PENDING的预约 |
| 表格列-访客姓名 | el-table-column | 显示 |
| 表格列-访客电话 | el-table-column | 显示 |
| 表格列-访客单位 | el-table-column | 显示 |
| 表格列-来访事由 | el-table-column | 显示 |
| 表格列-被访人 | el-table-column | 显示 |
| 表格列-被访人部门 | el-table-column | 显示 |
| 表格列-来访时间 | el-table-column | 日期格式化 |
| 表格列-操作 | el-table-column | 通过/拒绝按钮 |
| 审批通过按钮 | el-button | 点击弹出审批通过弹窗 |
| 审批拒绝按钮 | el-button | 点击弹出审批拒绝弹窗 |

### 审批通过弹窗

| 字段 | 类型 | 说明 |
|------|------|------|
| 预约信息 | 显示 | 显示访客姓名、来访时间等 |
| 审批备注 | el-input (textarea) | 可选（审批意见） |

### 审批拒绝弹窗

| 字段 | 类型 | 校验 |
|------|------|------|
| 预约信息 | 显示 | 显示访客信息 |
| 拒绝原因 | el-input (textarea) | **必填** |

## 4. 正向测试用例

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| APPROVE-001 | 待审批列表加载 | 进入审批管理页面 | 仅显示状态=PENDING的预约 |
| APPROVE-002 | 审批通过 | 1.点击"通过" 2.填写备注 3.确认 | 审批通过，状态变为APPROVED |
| APPROVE-003 | 审批通过-后续动作 | 审批通过后检查 | 1.自动创建VISITING记录 2.发送审批短信给访客 3.预约列表状态更新 |
| APPROVE-004 | 审批拒绝 | 1.点击"拒绝" 2.填写拒绝原因 3.确认 | 审批拒绝，状态变为REJECTED |
| APPROVE-005 | 审批拒绝-原因必填 | 1.点击拒绝 2.不填原因 3.确认 | 提示"拒绝原因不能为空" |
| APPROVE-006 | 审批后列表更新 | 审批一条后查看列表 | 该预约从待审批列表中移除 |
| APPROVE-007 | 空列表 | 所有预约都已审批后 | 显示空状态"暂无待审批记录" |

## 5. 短信通知测试

| 用例ID | 测试名称 | 步骤 | 预期结果 |
|--------|----------|------|----------|
| APPROVE-S01 | 审批通过短信 | 1.审批通过 2.查看短信日志 | 生成VISITOR_APPROVE类型的短信记录 |
| APPROVE-S02 | 短信内容变量 | 查看审批通过短信内容 | 变量全部替换：{visitorName}、{hostName}、{visitTime}、{passCode} |
| APPROVE-S03 | 通行码链接 | 查看短信中通行码链接 | 链接包含 `/pass/{passCode}` |

## 6. 边界与异常测试

| 用例ID | 测试名称 | 条件/步骤 | 预期结果 |
|--------|----------|-----------|----------|
| APPROVE-B01 | 重复审批 | 对已审批的预约再次审批 | 提示"该预约已被审批" |
| APPROVE-B02 | 审批已取消的预约 | 预约在打开页面后被他人在预约管理中取消 | 提示该预约状态已变更 |
| APPROVE-E01 | 无待审批记录 | 所有预约已审批或取消 | 显示空状态 |

## 7. 权限测试

| 用例ID | 条件 | 预期结果 |
|--------|------|----------|
| APPROVE-P01 | 无`visitor:appointment:pending` | 页面不可访问 |
| APPROVE-P02 | 无`visitor:appointment:approve` | 通过/拒绝按钮不显示 |
| APPROVE-P03 | 通行码公开访问 | 匿名访问 `/pass/{code}` | 显示访客信息，无需登录 |
