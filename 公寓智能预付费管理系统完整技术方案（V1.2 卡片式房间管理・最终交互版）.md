## 一、项目概述

本系统面向集中式公寓、长租公寓、员工宿舍，实现**房间楼栋管理、卡片式可视化房间管理、多租客合租管理、冷热水 / 电表计量计费、预付费 & 后付费双模式、多类型门锁管控、设备品牌‑型号‑指令统一管理、设备报修工单、用量排行与阈值预警、微信在线支付、合租全员欠费催费推送、AI 能耗智能异常分析**一体化解决方案。

核心业务约束：

1. 水费区分冷水、热水，独立计量、独立费率、独立余额。
2. 冷水余额不足 / 热水余额不足 / 电费余额不足 → **电表远程断电，不停水，门锁权限不受任何影响**。
3. 门锁分为联网智能锁、离线 NFC 锁两类，权限仅受租期控制，与水电欠费解耦。
4. 同一房间多人合租时，欠费、用量超标、租期到期消息**全员独立推送至每个租客微信公众号**。
5. 设备新增采用「品牌 → 型号 → 设备号」三级录入，自动带入预设通信指令，支持多厂商、多协议接入。
6. AI 能耗智能分析：实时监控水电用量、功率、用水曲线，智能识别漏水、漏电、违规大功率电器、偷电、夜间异常使用等风险，自动生成分析报告并预警推送。
7. **房间管理新增卡片可视化模式**，与传统表格模式并存，数据实时双向同步。

## 四、房间卡片视图 前端交互详细规划（严格按你 5 点要求）

### 4.1 卡片基础规格

1. **固定卡片尺寸**：统一宽 `280px`、高 `220px`，分辨率固定，不随内容拉伸；
2. **布局方式**：弹性网格布局，**自适应屏幕宽度自动调整每行卡片数量**（大屏 6 个、中屏 4 个、小屏 2 个）；
3. **视图切换**：房间模块支持【表格视图 / 卡片视图】一键切换，数据双向实时同步；
4. **禁用拖拽**：所有房间分配、入住、绑定操作**纯手工表单输入选择**，无拖拽交互；
5. **弹窗类型**：所有操作均使用**居中模态弹窗**，无右侧抽屉。

### 4.2 卡片配色规则（静态状态，无动态实时数据）

卡片底色根据房间状态固定配色，**仅展示静态状态标签，不显示实时余额、实时用量**：

- 绿色：空置可租
- 蓝色：正常入住
- 黄色：预警状态（用量超标 / 租期临近到期）
- 橙色：AI 能耗异常（漏水 / 漏电 / 违规电器）
- 红色：欠费断电
- 灰色：维修中 / 停用

### 4.3 卡片固定展示信息（严格仅展示静态信息，无动态实时数据）

卡片内从上至下固定展示，**不加载实时余额、实时功率、实时流量等动态数据**：

1. 顶部：**房间编号、户型、面积**（静态基础信息）
2. 右上角：状态角标（空置 / 入住 / 欠费 / 预警 / 异常 / 维修）
3. 中部静态区：
   - 所属公寓 - 楼栋 - 楼层
   - 当前合租人数（静态统计数）
   - 门锁类型（联网锁 / NFC 锁）
   - 设备绑定状态（已绑定 / 未绑定）
4. 底部：**租期起止时间（静态已录入时间）**
5. 卡片无任何实时动态数据、无实时曲线、无实时余额。

> 设计原则：卡片只做**状态概览**，详细数据、动态实时数据、操作全部在**居中弹窗**内完成。

### 4.4 核心交互流程（全部基于点击卡片 → 弹出居中操作弹窗）

#### 流程 1：查看房间详情

1. 点击任意房间卡片；
2. 弹出**居中全屏弹窗**，展示完整房间信息：租客、合租明细、设备绑定、费率、分摊规则、AI 分析、报修记录；
3. 弹窗内可切换标签页：基础信息｜租客管理｜设备绑定｜账单｜能耗分析｜报修。

#### 流程 2：租客入住（手工录入，无拖拽）

1. 点击【空置】绿色卡片；
2. 弹出居中入住弹窗；
3. 手工选择 / 新增租客、填写租期起止时间、选择合租分摊规则、选择付费模式；
4. 手工勾选分配 NFC 卡片 / 下发联网锁权限；
5. 提交后：
   - 房间状态更新为【已入住】；
   - 卡片自动变为蓝色；
   - 传统表格房间数据同步更新；
   - 租客、设备、权限同步生效。

#### 流程 3：退租结算

1. 点击已入住房间卡片；
2. 弹窗内点击【退租结算】；
3. 系统自动核算账单、余额，管理员手工确认；
4. 提交后卡片变为绿色空置，表格同步更新，门锁权限回收。

#### 流程 4：设备绑定（电表 / 冷热水表 / 门锁）

1. 点击房间卡片 → 弹窗内进入【设备绑定】标签；
2. 手工选择设备品牌 → 型号 → 设备 SN；
3. 批量或逐个绑定水电锁设备；
4. 卡片更新设备绑定状态，表格同步设备关联关系。

#### 流程 5：AI 异常、欠费处理

1. 橙色 / 红色卡片代表异常 / 欠费；
2. 点击卡片弹出弹窗，查看完整 AI 分析结论、异常原因；
3. 弹窗内可一键发起催费推送、报修、查看能耗曲线；
4. 处理完成后卡片状态自动刷新，表格同步。

### 4.5 筛选与批量操作

1. 顶部筛选栏：公寓、楼栋、楼层、房间状态、设备绑定状态；
2. 支持批量选中卡片，批量入住、批量退租、批量绑定设备；
3. 所有批量操作均通过**居中批量弹窗**手工确认执行。

### 4.6 数据同步机制（核心保障）

1. 卡片视图、传统表格视图**共用同一套后端接口与数据库**；
2. 任意一端修改房间信息、状态、租客、设备，另一端**实时自动刷新同步**；
3. 卡片仅做静态状态渲染，动态实时数据全部由弹窗内接口加载，不占用卡片性能。

字段：

3. 解析规则：读数倍率、小数位数、字节偏移、数据格式。

4. 备注。

   

   功能：新增、编辑、删除、指令测试、批量导入导出。

## 七、AI 能耗智能分析模块（核心新增，模拟 AI 判断）

### 7.1 后台可配置 AI 分析规则（直接可视化配置）

1. 用电风险规则

   - 瞬时功率＞2000W：判定使用大功率违规电器（热得快、电磁炉、电暖器）
   - 夜间 23:00‑06:00 持续大功率用电：异常熬夜用电、私拉电线风险
   - 长期不用电却持续微量耗电：漏电、偷电
   - 单日用电量远超同户型 3 倍以上：异常用电

2. 用水风险规则

   - 24 小时持续瞬时流量＞5L/h：水管漏水、马桶长流水、水龙头未关
   - 长期无人入住仍持续用水：漏水风险
   - 单日用水量远超同户型 2 倍：异常用水

3. 基线学习

   

   系统自动学习房间 7 天历史正常用量基线，超过基线 2 倍自动预警。

4. 时段分析

   

   区分白天、夜间、工作日、周末，识别异常时段使用行为。

### 7.2 AI 自动分析输出结果（弹窗内展示，卡片仅标异常状态）

系统自动生成结构化分析文案示例：

- “该房间瞬时功率 2350W，疑似使用热得快 / 电磁炉等违规大功率电器，存在安全隐患”
- “冷水 24 小时持续有流量，疑似管道漏水、马桶长流水，请及时排查”
- “夜间 0‑5 点持续高功率用电，疑似私拉电线、违规用电”
- “用电量较往期上涨 320%，远超同户型平均水平，存在偷电或大功率设备使用风险”
- “长期无人入住仍有持续用水，漏水风险极高”

### 7.3 预警推送

异常触发后：

1. 房间卡片标为**橙色异常状态**；
2. 居中弹窗内展示完整分析报告；
3. 管理员收到系统告警；
4. 租客公众号推送能耗异常提醒；
5. 可配置：严重异常自动断电。

## 八、合租水电费分摊算法（4 种可配置）

### 8.1 方案 1：人数平均分摊（默认）

单人费用 = 房间总费用 ÷ 有效合租人数

### 8.2 方案 2：入住天数分摊（公平优先）

单人费用 = 房间总费用 ×（个人本期在住天数 ÷ 所有租客总在住天数）

### 8.3 方案 3：床位权重分摊（宿舍公寓）

单人费用 = 房间总费用 ×（个人床位权重 ÷ 总权重和）

### 8.4 方案 4：独立账户分摊（强管控）

单人独立扣费，可配置单人欠费即整间断电。

### 8.5 预付费模式

每日定时抄表 → 按规则分摊扣费 → 任一租客余额不足 → 全员推送催费 → 电表断电。

### 8.6 后付费模式

按月抄表 → 生成个人账单 → 推送账单 → 逾期未缴 → 全员催费 → 断电。

### 9.3 预警与推送规则

十三、定时任务核心逻辑（Java 伪代码，含 AI 分析）

```java
// 1. 抄表获取实时数据
BigDecimal power = getRealTimePower(meterSn); // 瞬时功率
BigDecimal flow = getRealTimeFlow(waterSn);   // 瞬时流量

// 2. AI规则判断
String analysisResult = "";
boolean isAbnormal = false;

// 违规大功率电器判断
if(power.compareTo(new BigDecimal("2000")) > 0){
    analysisResult = "该房间瞬时功率超过2000W，疑似使用热得快、电磁炉等违规大功率电器，存在安全隐患";
    isAbnormal = true;
}

// 漏水判断
if(flow.compareTo(new BigDecimal("5")) > 0 && isAllDayFlow(waterSn)){
    analysisResult = "冷水24小时持续有流量，疑似管道漏水、马桶长流水、水龙头未关，请及时排查";
    isAbnormal = true;
}

// 夜间异常用电
if(isNightTime() && power.compareTo(new BigDecimal("1000")) > 0){
    analysisResult = "夜间0‑5点持续高功率用电，疑似私拉电线、违规用电";
    isAbnormal = true;
}

// 3. 异常则推送告警，更新房间静态状态（卡片同步变色）
if(isAbnormal){
    // 更新房间状态为AI异常，同步卡片与表格
    updateRoomAbnormalStatus(roomId);
    List<Tenant> tenantList = getValidTenantByRoom(roomId);
    List<String> openIds = tenantList.stream().map(Tenant::getOpenId).collect(Collectors.toList());
    wechatPushService.batchSend(openIds, "【能耗异常提醒】" + analysisResult);
    saveAiLog(roomId, analysisResult);
}

// 4. 欠费判断逻辑不变
boolean roomOwe = checkRoomOwe(roomId);
if(roomOwe){
    updateRoomOweStatus(roomId); // 卡片变红
    List<Tenant> tenantList = getValidTenantByRoom(roomId);
    List<String> openIds = tenantList.stream().map(Tenant::getOpenId).collect(Collectors.toList());
    wechatPushService.batchSend(openIds, "您所在房间水电余额不足，请及时充值，即将断电！");
    DeviceModel model = getModelByMeter(meterId);
    mqttClient.send(model.getPowerOffCmd(), meterSn);
}
```

## 十四、部署与对接说明

1. 后台：RuoYi‑Vue 前后端分离部署，集成 MQTT 客户端、微信支付 SDK、消息推送 SDK、AI 能耗分析引擎；**仅新增房间卡片视图前端页面，后端接口、数据库完全复用**。
2. 公众号：微信认证服务号，配置 JSAPI 支付、模板消息、获取 OpenID 权限。
3. 设备：水电表采用 NB‑IoT MQTT 协议，必须支持瞬时功率、瞬时流量上报；门锁支持联网 MQTT 或离线 NFC。
4. 定时任务：每日执行抄表、扣费、用量统计、排行计算、预警检测、消息推送、AI 能耗实时分析。
5. 扩展性：新增设备型号仅需后台配置指令，AI 规则后台可视化配置，无需修改代码。
