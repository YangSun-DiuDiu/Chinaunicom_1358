# 智慧园区驾驶舱重构 — 设计文档

> 版本：v1.0 | 日期：2026-07-01 | 状态：已确认

## 1. 目标

将现有首页从"设备/访客双模式切换"重构为**统一4模块驾驶舱**，数据全部对接真实业务，深色科技蓝主题。

## 2. 布局方案

采用**方案三：顶排KPI + 中排图表网格 + 底排动态列表**

```
┌─────────────────────────────────────────────────────┐
│  TOP BAR: 4 大 KPI 卡片（单行）                       │
│  [🖥 设备]  [👤 访客]  [📅 会议]  [🏠 公寓]          │
├─────────────────────────────────────────────────────┤
│  MIDDLE: 2×2 图表网格                                │
│  [设备类型饼图]  [访客状态柱状图]                      │
│  [公寓状态环形图] [今日会议列表]                       │
├─────────────────────────────────────────────────────┤
│  BOTTOM: 动态列表（2列）                              │
│  [最新来访 5行]  [最近事件/告警 5行]                   │
└─────────────────────────────────────────────────────┘
```

## 3. 配色方案

| 元素 | 颜色 | 说明 |
|------|------|------|
| 页面底色 | `#0a1a2e` | 深海军蓝 |
| 卡片底色 | `rgba(13,33,55,0.85)` | 半透明蓝黑 |
| 卡片边框 | `rgba(0,180,255,0.12)` | 微光边框 |
| KPI 数字 | `#00d4ff` | 电光蓝 + glow |
| KPI 标签 | `#8899aa` | 灰蓝 |
| 告警色 | `#ff5252` / `#ff9100` | 红/橙 |
| ECharts 主题 | `dark` | 暗色主题 |

## 4. 数据映射

### 4.1 KPI 卡片

| 模块 | 主数据 | 副数据 | API |
|------|--------|--------|-----|
| 🖥 IoT设备 | 总数 X | 在线X / 离线X(⚠) | `GET /dashboard/device` |
| 👤 访客 | 今日 X 人 | 待审批 X(⚠) | `GET /dashboard/visitor` |
| 📅 会议 | 总预约 X 场 | 今日 X 场 | `GET /dashboard/meeting` **(NEW)** |
| 🏠 公寓 | 房源 X 间 | 入住X / 空闲X | `GET /dashboard/apartment` **(NEW)** |

### 4.2 图表数据

| 图表 | 类型 | 数据来源 |
|------|------|---------|
| 设备类型分布 | 饼图 (ring) | `deviceTypeStats` ← 已有API |
| 访客状态分布 | 柱状图 (bar) | `visitStats` ← 已有API |
| 公寓状态分布 | 环形图 (pie) | 新房源状态统计 ← **NEW API** |
| 今日会议 | 列表/时间线 | 今日会议列表 ← **NEW API** |

### 4.3 底部列表

| 列表 | 行数 | 数据来源 |
|------|------|---------|
| 最新来访 | 5 | `recentVisits` ← 已有API |
| 最近事件 | 5 | 合并告警+心跳 ← 已有API |

## 5. 后端改动

### 5.1 新增 API

**GET `/dashboard/meeting`**
```json
{
  "totalBookings": 8,
  "todayBookings": 2,
  "roomCount": 3,
  "todayMeetings": [
    {"meetingName": "...", "roomName": "...", "startTime": "...", "endTime": "...", "status": "..."}
  ]
}
```

**GET `/dashboard/apartment`**
```json
{
  "totalRooms": 5,
  "statusDistribution": [
    {"status": "GREEN", "label": "空置", "count": 2},
    {"status": "CYAN", "label": "部分入住", "count": 1},
    {"status": "BLUE", "label": "满房", "count": 1},
    {"status": "GRAY", "label": "维修", "count": 1}
  ],
  "occupiedCount": 2,
  "vacantCount": 3
}
```

## 7. 公寓房间状态规则（复用）

| 条件 | 状态 | 颜色 |
|------|------|------|
| actual=0 | GREEN | 绿色(空置) |
| 0<actual<capacity | CYAN | 青色(部分入住) |
| actual>=capacity | BLUE | 蓝色(满房) |
| 手动设置 | GRAY | 灰色(维修) |

## 8. 改动范围

- 新增后端 API: 2 个
- 新增前端 API 函数: 2 个
- 重写前端页面: 1 个 (`views/index.vue`)
- 不影响的文件: 设备驾驶舱/访客驾驶舱旧页面可以保留为备用

## 9. 验收标准

- [ ] 页面整体深蓝底色 #0a1a2e，无白色背景
- [ ] 4 个 KPI 卡片数据全部来自真实 API
- [ ] 4 个图表格子正确渲染（2饼图+1柱状+1列表）
- [ ] 底部最新来访/最近事件各 5 行
- [ ] 所有数字通过 API 动态加载，无硬编码
