## 一、系统概述

### 1.1 项目背景

智慧园区管理系统是一套面向产业园区、写字楼、学校等场景的综合管理平台，集成了**设备监控、访客管理、维修工单、短信通知**等核心功能，帮助园区运营方实现数字化管理。

### 1.2 技术架构

| 层级 | 技术 |
|------|------|
| 前端框架 | Vue 2.x + Element UI |
| 后端框架 | Spring Boot + MyBatis |
| 安全认证 | Spring Security + JWT |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis |
| Web服务器 | Nginx |
| 定时任务 | Quartz |

### 1.3 功能模块总览

```
智慧园区管理系统
├── 🏠 首页驾驶舱     设备/访客双模式数据大屏
├── 👥 人员管理       用户、角色、部门管理
├── 🖥️ 设备管理       设备列表、拓扑、心跳、故障、SNMP、维修工单
├── 🚶 访客管理       预约、审批、现场登记、日志、H5自助登记
├── 💬 短信管理       阿里云短信配置、发送记录
├── 📋 日志管理       操作日志、登录日志
├── ⚙️ 系统管理       菜单配置
└── 👤 个人中心       资料、密码、头像
```

> 📸 **截图**: 系统登录页面，展示"智慧园区管理系统"标题、用户名/密码/验证码登录表单

## 四、设备管理

设备管理是本系统的核心模块，包含6个子功能。

### 4.1 设备列表

| 功能 | 说明 |
|------|------|
| **列表查询** | 按设备名称、设备类型（网络/监控/NVR/其他）、状态（在线/离线/未知）筛选 |
| **新增设备** | 填写设备名称、类型、IP、型号、位置、SNMP配置、责任人等 |
| **编辑设备** | 修改设备信息和配置 |
| **删除设备** | 单个/批量删除 |
| **导出设备** | Excel导出，按当前筛选条件 |
| **设备详情** | 点击设备名查看完整信息 |
| **报修** | 离线设备出现"报修"按钮，一键创建维修工单并发送短信 |
| **查看端口** | 查看设备端口列表 |

**设备字段说明**：

| 字段 | 说明 |
|------|------|
| 设备名称 | 必填，设备唯一标识 |
| 设备类型 | NETWORK(网络)、MONITOR(监控)、NVR(录像机)、OTHER(其他) |
| IP地址 | 必填，用于 Ping 心跳检测 |
| SNMP配置 | 团体名、端口(默认161)、版本(v2c/v3) |
| 巡检周期 | 1-10分钟，心跳检测间隔 |
| 负责人/电话 | 故障时接收短信通知 |

### 4.6 维修工单 🆕

**V6新增**完整的设备维修工单流转系统：

#### 工单状态流转

```
PENDING(待处理) ──转派──▶ ASSIGNED(已派发) ──接收──▶ ACCEPTED(已接收) ──确认──▶ COMPLETED(已完成)
                              │                                    │
                              └──拒绝──▶ REJECTED(已拒绝) ──退回──┘
```

#### 自动创建机制

| 触发方式 | 说明 |
|----------|------|
| **心跳检测离线** | 设备被 Ping 不通 → 自动创建工单 → 发短信给负责人 |
| **手动报修** | 设备列表页点击"报修"按钮 → 创建工单 → 发短信 |

#### 转派功能

| 操作 | 说明 |
|------|------|
| **转派** | 当前责任人将工单转给他人，填写被转派人姓名+电话+原因 |
| **接收** | 被转派人确认接收，工作量转移 |
| **拒绝** | 被转派人拒绝，填写原因，工单退回原责任人 |

#### 短信维修确认

责任人收到的短信包含维修确认链接：

```
【设备离线告警】设备「交换机1」（IP:192.168.1.1）已离线，请及时处理。
维修完成后点击确认: http://1.94.26.126:3000/repair-complete?token=abc123...
```

点击链接打开 **H5维修确认页（无需登录）**，维护人员可直接确认维修完成并填写结果。

> 📸 **截图**: 维修工单列表页面，展示工单状态流转和操作按钮（转派/接收/拒绝）

#### 工作量统计

独立页面按维护人员维度统计：

| 指标 | 说明 |
|------|------|
| 总工单数 | 该维护人员所有工单 |
| 已完成 | COMPLETED 状态工单数 |
| 进行中 | PENDING + ASSIGNED + ACCEPTED |
| 已拒绝 | REJECTED 状态工单数 |
| 完成率 | 已完成 / 总工单 × 100%（进度条） |

> 📸 **截图**: 工作量统计页面，表格展示每位维护人员的工单数据，含完成率进度条

| 功能 | 说明 |
|------|------|
| **列表查询** | 按访客姓名、电话、状态、来访时间筛选 |

```
PENDING(待审批) ──审批通过──▶ APPROVED(已通过) ──进入──▶ VISITING(来访中) ──离开──▶ COMPLETED(已完成)
     │                         │                              │
     ├──审批拒绝──▶ REJECTED   ├──取消──▶ CANCELLED          │
     └──取消──▶ CANCELLED                                     │
```

## 十、权限体系

### 10.1 权限模型

采用 **RBAC（角色-菜单-权限）** 模型：

```
用户 ──┬── 角色 ──┬── 菜单权限（页面访问 + 按钮操作）
       │           │
       │           └── 数据权限（全部/自定义/本部门/本部门及以下/仅本人）
       │
       └── 岗位
```

### 10.2 权限控制层次

| 层次 | 实现方式 | 说明 |
|------|----------|------|
| **路由控制** | `route.meta.permissions` | 无权限的路由不注册，侧边栏不显示 |
| **按钮控制** | `v-hasPermi` 指令 | 无权限按钮从 DOM 中移除 |
| **API控制** | `@PreAuthorize("@ss.hasPermi(...)")` | 后端接口层鉴权 |
| **数据权限** | `@DataScope` 注解 | 按角色数据范围过滤查询结果 |

### 10.3 权限标识示例

| 模块 | 权限标识 | 说明 |
|------|----------|------|
| 用户管理 | `system:user:list / add / edit / remove` | 列表/新增/编辑/删除 |
| 设备管理 | `device:list / device:add / device:repair` | 列表/新增/创建工单 |
| 维修工单 | `device:repair:list / transfer / accept / reject` | 列表/转派/接收/拒绝 |
| 访客管理 | `visitor:appointment:list / approve / cancel` | 列表/审批/取消 |
| 短信管理 | `sms:config:edit / sms:log:list` | 配置编辑/日志查看 |
