# 01 — 项目概述

## 1. 项目定位

本项目是基于 **RuoYi-Vue（若依前后端分离版）** 深度定制开发的 **智慧园区管理系统（Smart Park Management System）**。

在原若依 RBAC 权限体系基础上，扩展了以下业务模块：
- **设备管理** — IoT 设备全生命周期管理、心跳监测、故障告警、SNMP 监控
- **访客管理** — 预约审批、现场登记、H5 自助登记、通行凭证
- **智能化管理** — 人车出入控制、视频监控、电子地图
- **考勤管理** — 对接钉钉/企业微信打卡回调
- **短信管理** — 阿里云短信集成（设备告警、访客通知）

## 3. 部署架构

```
用户浏览器 (Vue2 SPA)
    │
    ├── /prod-api/* ──→ Nginx 反向代理 ──→ SpringBoot (8080)
    │                                         │
    ├── /ws/* (WebSocket) ────────────────────┤ (考勤推送、视频流)
    │                                         │
    └── 静态资源 (Nginx直接返回)              ├── MySQL 8.x (ry101)
                                              ├── Redis 7.x (Token/缓存)
                                              └── 钉钉/企业微信 Open API
```

**部署模式**：
- 开发环境：前端 `vue-cli-service serve` 代理到 `localhost:8080`
- 生产环境：前端打包为静态文件由 Nginx 提供，后端独立 JAR 运行
- 也支持 WAR 部署到外部 Tomcat（通过 `RuoYiServletInitializer`）

## 5. 关键设计决策

1. **全静态路由** — 废弃若依动态路由机制，路由在前端 `staticRoutes` 中预先定义，登录后按权限过滤
2. **无状态 JWT** — 不使用 HTTP Session，Token 存 Cookie，Redis 存储 LoginUser
3. **Quartz 内存模式** — 注释掉 JDBC JobStore，使用 RAMJobStore，启动时从 `sys_job` 表加载
4. **读写分离支持** — DynamicDataSource 支持主从切换（从库默认关闭）
5. **WebSocket 双通道** — 考勤实时推送 + 视频流实时转发
