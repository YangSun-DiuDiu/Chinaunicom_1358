# 09 — 工作流程与数据流

## 1. 用户登录认证流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户打开 /login 页面                                       │
│    ↓                                                         │
│ 2. 前端 GET /captchaImage                                    │
│    → CaptchaController 生成验证码 → 存 Redis (2分钟) → 返回 Base64 图片 + UUID │
│    ↓                                                         │
│ 3. 用户输入 用户名 + 密码 + 验证码                               │
│    ↓                                                         │
│ 4. 前端 RSA 加密密码 → POST /login                            │
│    → SysLoginService.login()                                 │
│      ├── 验证码校验 (Redis CAPTCHA_CODE_KEY + uuid)            │
│      ├── 用户名/密码长度校验                                   │
│      ├── IP 黑名单检查                                        │
│      ├── AuthenticationManager.authenticate()                │
│      │   → UserDetailsServiceImpl.loadUserByUsername()       │
│      │     ├── 查 SysUser                                    │
│      │     ├── 检查删除/禁用                                   │
│      │     ├── 密码重试锁定检查 (SysPasswordService)           │
│      │     ├── BCrypt 密码校验                                │
│      │     ├── 加载菜单权限 (admin → *:*:*, 其他 → 查DB)       │
│      │     └── 返回 LoginUser (UserDetails)                  │
│      ├── TokenService.createToken(loginUser)                 │
│      │   ├── 生成 fastUUID                                    │
│      │   ├── 创建 JWT (HS512, claims: userId+username+... )   │
│      │   └── Redis 存储 LoginUser (LOGIN_TOKEN_KEY + uuid)    │
│      ├── AsyncFactory.recordLogininfor() → 异步记录登录日志     │
│      └── 返回 { token: "Bearer xxx" }                        │
│    ↓                                                         │
│ 5. 前端 setToken(token) → Cookie.set('Admin-Token', token)   │
│    ↓                                                         │
│ 6. 前端 store.dispatch('GetInfo') → GET /getInfo             │
│    → 返回 { user, roles, permissions }                       │
│    ↓                                                         │
│ 7. 前端 store.dispatch('GenerateRoutes')                     │
│    → 按 permissions 过滤 staticRoutes → router.addRoutes()    │
│    → 跳转首页 /index                                          │
└─────────────────────────────────────────────────────────────┘
```

## 2. 请求认证流程（已登录）

```
┌─────────────────────────────────────────────────────────────┐
│ 客户端发送请求 (Authorization: Bearer <token>)                 │
│    ↓                                                         │
│ CorsFilter (跨域处理)                                         │
│    ↓                                                         │
│ JwtAuthenticationTokenFilter (OncePerRequestFilter)          │
│    ├── 提取 Authorization Header                             │
│    ├── TokenService.getLoginUser(request)                    │
│    │   ├── 解析 JWT → 获取 UUID                              │
│    │   ├── Redis 查找 LOGIN_TOKEN_KEY + uuid                 │
│    │   └── 返回 LoginUser                                   │
│    ├── 验证 Token 有效期                                      │
│    │   └── 距过期 < 20分钟 → 自动续期 (刷新 Redis TTL + JWT)   │
│    └── 注入 SecurityContext: authentication = LoginUser      │
│    ↓                                                         │
│ RepeatSubmitInterceptor                                      │
│    ├── 检查 @RepeatSubmit 注解                                │
│    └── Redis 对比 URL+参数+Token → 防重复提交                  │
│    ↓                                                         │
│ Controller 方法                                              │
│    ├── @PreAuthorize("hasPermission('xxx:xxx:list')")        │
│    │   → PermissionService.hasPermi()                       │
│    ├── @Log → LogAspect (异步记录操作日志)                     │
│    ├── @DataScope → DataScopeAspect (注入数据权限SQL)          │
│    ├── @RateLimiter → RateLimiterAspect (Redis+Lua限流)       │
│    └── @DataSource → DataSourceAspect (主从切换)              │
│    ↓                                                         │
│ Service → Mapper → MyBatis XML → Druid → MySQL              │
│    ↓                                                         │
│ 返回 AjaxResult (JSON)                                       │
│    ↓                                                         │
│ 异常时 → GlobalExceptionHandler → AjaxResult(code, msg)      │
└─────────────────────────────────────────────────────────────┘
```

## 3. 权限控制流程

```
┌─────────────────────────────────────────────────────────────┐
│ 三层权限控制                                                   │
│                                                               │
│ 第一层: 路由级 (前端 permission.js)                            │
│   GET /getInfo → roles[], permissions[]                      │
│   → 遍历 staticRoutes                                       │
│   → 每个路由的 meta.permissions 检查 $auth.hasPermiOr()       │
│   → 无权限的路由不加入 router                                 │
│                                                               │
│ 第二层: UI级 (前端 v-hasPermi / v-hasRole 指令)               │
│   按钮/菜单/操作项根据权限字符串显示/隐藏                         │
│   v-hasPermi="['system:user:add']"                           │
│   → 查 store.user.permissions 数组                           │
│   → 无权限 → element.parentNode.removeChild(element)         │
│                                                               │
│ 第三层: 接口级 (后端 @PreAuthorize)                             │
│   @PreAuthorize("@ss.hasPermi('system:user:list')")          │
│   → PermissionService.hasPermi('system:user:list')           │
│   → 查当前 LoginUser.permissions 集合                         │
│   → *:*:* 通配 → 放行                                        │
│   → 无权限 → AccessDeniedException → 403                     │
└─────────────────────────────────────────────────────────────┘
```

## 4. 数据权限流程

```
┌─────────────────────────────────────────────────────────────┐
│ Controller 方法标注 @DataScope(deptAlias="d", userAlias="u")  │
│    ↓                                                         │
│ DataScopeAspect @Before 切入                                 │
│    ├── 获取当前用户 LoginUser                                  │
│    ├── 获取用户的角色列表                                       │
│    ├── 遍历角色，按 data_scope 拼装 SQL:                        │
│    │   DATA_SCOPE_ALL (1)          → 不添加条件               │
│    │   DATA_SCOPE_CUSTOM (2)       → AND d.dept_id IN (       │
│    │        SELECT dept_id FROM sys_role_dept                 │
│    │        WHERE role_id = ?)                                │
│    │   DATA_SCOPE_DEPT (3)         → AND d.dept_id = ?        │
│    │   DATA_SCOPE_DEPT_AND_CHILD (4)→ AND (d.dept_id = ? OR   │
│    │        d.dept_id IN (子部门列表))                          │
│    │   DATA_SCOPE_SELF (5)         → AND u.user_id = ?        │
│    │   多个角色 → OR 连接                                       │
│    ├── 注入 params.dataScope                                  │
│    └── MyBatis XML: SELECT ... WHERE ... ${params.dataScope}  │
└─────────────────────────────────────────────────────────────┘
```

## 5. CRUD 业务开发完整流程（单表示例）

```
以新增业务模块 "设备类型管理" 为例：

步骤1: 数据库
  CREATE TABLE iot_device_type (
    type_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    type_name VARCHAR(100) NOT NULL COMMENT '类型名称',
    type_code VARCHAR(50) COMMENT '类型编码',
    status CHAR(1) DEFAULT '0' COMMENT '状态',
    create_by VARCHAR(64),
    create_time DATETIME,
    update_by VARCHAR(64),
    update_time DATETIME,
    del_flag CHAR(1) DEFAULT '0',
    remark VARCHAR(500)
  );

步骤2: 后端 Domain (ruoyi-system)
  DeviceType extends BaseEntity → typeId, typeName, typeCode, status

步骤3: 后端 Mapper (ruoyi-system)
  DeviceTypeMapper.java (接口)
  DeviceTypeMapper.xml (XML → resultMap, selectList, insert, update, delete)

步骤4: 后端 Service (ruoyi-system)
  IDeviceTypeService.java (接口)
  DeviceTypeServiceImpl.java (实现, @Service, @Transactional)

步骤5: 后端 Controller (ruoyi-admin)
  DeviceTypeController.java
    → @RestController, @RequestMapping("/device/type")
    → @PreAuthorize 权限控制
    → list()/getInfo()/add()/edit()/remove()/export()

步骤6: 前端 API (ruoyi-ui)
  src/api/device/type.js
    → getList, getType, addType, updateType, delType, exportType

步骤7: 前端页面 (ruoyi-ui)
  src/views/device/type/index.vue
    → 搜索区 (el-form) + 表格 (el-table) + 分页 (Pagination)
    → 新增/修改弹窗 (el-dialog + el-form + rules 校验)
    → 导出按钮 + 导入按钮 + 刷新

步骤8: 菜单权限 SQL
  INSERT INTO sys_menu VALUES (..., '设备类型管理', 'device:type:list', ...);
  INSERT INTO sys_menu VALUES (..., '设备类型新增', 'device:type:add', ...);
  INSERT INTO sys_role_menu VALUES (...);
```

## 6. WebSocket 实时通信流程

### 考勤推送

```
钉钉/企业微信服务器
    │
    │ POST /attendance/callback (打卡事件通知)
    ↓
AttendanceCallbackController
    │
    ├── 解析回调数据 → 存入数据库
    │
    └── AttendanceWebSocketHandler.broadcast(json)
         │
         ├── 遍历所有已连接 WebSocket Session
         └── session.sendMessage(new TextMessage(json))
              ↓
         前端 /ws/attendance WebSocket 客户端
              ↓
         实时更新考勤界面
```

### 视频流

```
前端 VideoStreamController
    │
    │ WebSocket /ws/video/{deviceId}
    ↓
VideoWebSocketHandler
    │
    │ 管理 viewer 集合 (ConcurrentHashMap<deviceId, Set<WebSocketSession>>)
    │
    │ 设备帧数据 → broadcast(deviceId, frame)
    │   → 遍历该设备的所有 viewer
    │   → viewer.sendMessage(frame)
    ↓
前端 Canvas/Video 渲染
```
