# 02 — 模块架构

## 1. 多模块结构

```
ruoyi (根 POM, packaging=pom)
 │
 ├── ruoyi-common         基础设施层 — 无业务依赖，被所有模块依赖
 │   ├── annotation/      9 个自定义注解 (@DataScope, @Log, @Excel, @RateLimiter...)
 │   ├── core/            基础类 (BaseEntity, AjaxResult, BaseController...)
 │   ├── enums/           8 个枚举 (BusinessType, DataSourceType, UserStatus...)
 │   ├── exception/       异常体系 (20+ 异常类)
 │   ├── filter/          过滤器 (XSS, Repeatable, Referer)
 │   ├── constant/        常量 (Constants, CacheConstants, UserConstants...)
 │   ├── utils/           工具类 (String, File, Excel, IP, Spring, UUID...)
 │   └── xss/             XSS 校验注解
 │
 ├── ruoyi-framework      框架核心层 — 依赖 ruoyi-system + ruoyi-common
 │   ├── aspectj/         4 个切面 (Log, DataScope, DataSource, RateLimiter)
 │   ├── config/          14 个配置类 (Security, Druid, Redis, Captcha, MyBatis...)
 │   ├── datasource/      动态数据源 (主从切换)
 │   ├── interceptor/     拦截器 (防重复提交)
 │   ├── manager/         管理器 (异步任务、优雅关闭)
 │   ├── security/        安全 (JWT Filter, 认证/登出处理)
 │   └── web/             异常处理、权限服务、Token服务
 │
 ├── ruoyi-system         业务系统层 — 依赖 ruoyi-common
 │   ├── domain/          24 个实体类
 │   ├── mapper/          24 个 Mapper 接口
 │   ├── service/         30+ Service 接口与实现
 │   └── resources/mapper/system/  24 个 MyBatis XML
 │
 ├── ruoyi-quartz         定时任务层 — 依赖 ruoyi-system + ruoyi-common
 │   ├── domain/          SysJob, SysJobLog
 │   ├── mapper/          SysJobMapper, SysJobLogMapper
 │   ├── service/         任务调度服务
 │   ├── task/            自定义任务 (DeviceHeartbeatTask)
 │   └── util/            Quartz 工具 (AbstractQuartzJob, ScheduleUtils...)
 │
 └── ruoyi-admin          Web 入口层 — 依赖 ruoyi-framework + ruoyi-quartz
     ├── RuoYiApplication.java   启动类
     ├── web/controller/  40+ Controller (system/device/visitor/smart/attendance...)
     └── resources/       配置文件、静态资源
```

## 2. 模块依赖图

```
ruoyi-admin (启动入口)
  ├──→ ruoyi-framework (框架核心)
  │     ├──→ ruoyi-system (业务系统)
  │     │     └──→ ruoyi-common (通用工具)
  │     └──→ ruoyi-common
  └──→ ruoyi-quartz (定时任务)
        ├──→ ruoyi-system
        └──→ ruoyi-common
```

**依赖原则**：上层依赖下层，下层不感知上层；common 为零依赖基础层。

## 3. 分层架构（纵向）

```
┌─────────────────────────────────────────────────────────────┐
│ ruoyi-admin (Web 入口层)                                     │
│  Controller → @PreAuthorize 权限控制 → 参数绑定 → 返回统一格式  │
├─────────────────────────────────────────────────────────────┤
│ ruoyi-framework (切面/拦截层)                                 │
│  JWT Filter → LogAspect → DataScopeAspect → RateLimiter     │
│  → GlobalExceptionHandler → 统一异常兜底                     │
├─────────────────────────────────────────────────────────────┤
│ ruoyi-system (业务服务层)                                     │
│  Service Interface → ServiceImpl (@Transactional)            │
│  → Mapper Interface → MyBatis XML → Druid → MySQL            │
├─────────────────────────────────────────────────────────────┤
│ ruoyi-common (基础设施层)                                     │
│  BaseEntity → AjaxResult → RedisCache → ExcelUtil           │
│  → SecurityUtils → SpringUtils → StringUtils                │
└─────────────────────────────────────────────────────────────┘
```

## 4. 请求处理完整链路

```
HTTP Request
  │
  ├── CorsFilter                    (跨域处理)
  ├── JwtAuthenticationTokenFilter  (JWT 解析 → Redis 查 LoginUser → 注入 SecurityContext)
  ├── RepeatSubmitInterceptor       (防重复提交检测)
  ├── LocaleChangeInterceptor       (国际化)
  │
  ├── Controller 方法
  │     ├── @Log → LogAspect        (操作日志记录)
  │     ├── @DataScope → DataScopeAspect (数据权限 SQL 注入)
  │     ├── @RateLimiter → RateLimiterAspect (限流)
  │     └── @DataSource → DataSourceAspect (主从切换)
  │
  ├── Service 层 (业务逻辑)
  │     └── @Transactional
  │
  └── GlobalExceptionHandler        (异常统一转换为 AjaxResult)
       └── JSON Response
```
