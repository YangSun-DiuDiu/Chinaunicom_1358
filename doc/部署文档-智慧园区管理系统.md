## 1. 系统架构

```
                    ┌──────────────────────────┐
  浏览器            │     Nginx (port 80)      │
  http://IP  ─────→ │                            │
                    │  /          → 前端静态文件    │
                    │  /prod-api/ → 后端API代理   │
                    │  /ws/       → WebSocket代理 │
                    └──────────┬───────────────┘
                               │ proxy_pass
                    ┌──────────▼───────────────┐
                    │   Spring Boot (port 8080) │
                    │   ruoyi-admin.jar         │
                    └──┬──────────┬────────────┘
                       │          │
              ┌────────▼──┐  ┌───▼──────────┐
              │  MySQL 8  │  │  Redis 5.8   │
              │ (port 3306)│  │ (port 6379)  │
              └───────────┘  └──────────────┘
```

### 端口清单

| 端口 | 服务 | 说明 |
|------|------|------|
| 80 | Nginx | 前端 + API反向代理 |
| 8080 | Spring Boot | 后端Java服务 |
| 3306 | MySQL | 数据库 |
| 6379 | Redis | 缓存 + Session |

## 4. 数据库部署

### 4.1 安装 MySQL 8.0

```bash
sudo apt-get install -y mysql-server
sudo systemctl enable mysql
sudo systemctl start mysql

# 验证
mysql --version
```

### 4.2 创建数据库和用户

```sql
-- 登录MySQL
sudo mysql

-- 创建数据库
CREATE DATABASE IF NOT EXISTS ry101 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER IF NOT EXISTS 'sadmin'@'%' IDENTIFIED BY 'Chinaunicom@1358';
CREATE USER IF NOT EXISTS 'sadmin'@'localhost' IDENTIFIED BY 'Chinaunicom@1358';

-- 授权
GRANT ALL PRIVILEGES ON ry101.* TO 'sadmin'@'%';
GRANT ALL PRIVILEGES ON ry101.* TO 'sadmin'@'localhost';
FLUSH PRIVILEGES;

-- 允许远程访问（如需要）
-- 编辑 /etc/mysql/mysql.conf.d/mysqld.cnf
-- 修改 bind-address = 0.0.0.0
```

### 4.3 导入数据库结构

按以下顺序依次导入 SQL 文件：

```bash

mysql -u sadmin -pChinaunicom@1358 ry101 < ry_20260417.sql

# 2. Quartz 定时任务表

# 3. 业务迁移表（核心）
mysql -u sadmin -pChinaunicom@1358 ry101 < ry-vue100-migration.sql

# 4. 功能扩展表（按版本顺序）
mysql -u sadmin -pChinaunicom@1358 ry101 < ry-vue100-v7-features.sql
mysql -u sadmin -pChinaunicom@1358 ry101 < ry-vue100-v8-attendance-callback.sql
mysql -u sadmin -pChinaunicom@1358 ry101 < ry-vue100-v9-attendance-app.sql
mysql -u sadmin -pChinaunicom@1358 ry101 < ry-vue100-v10-org-sync.sql
```

## 6. 后端部署

### 6.1 准备部署目录

```bash
sudo mkdir -p /opt/ruoyi
```

### 6.2 上传 JAR 文件

将本地构建好的 `ruoyi-admin.jar` 上传到服务器：

```bash
scp ruoyi-admin/target/ruoyi-admin.jar root@192.168.1.60:/opt/ruoyi-admin.jar
```

### 6.3 创建配置文件

```bash
cat > /opt/ruoyi/application-druid.yml << 'YEOF'
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driverClassName: com.mysql.cj.jdbc.Driver
    druid:
      master:
        url: jdbc:mysql://127.0.0.1:3306/ry101?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=GMT%2B8
        username: sadmin
        password: Chinaunicom@1358
      slave:
        enabled: false
      initialSize: 5
      minIdle: 10
      maxActive: 20
  data:
    redis:
      host: 127.0.0.1
      port: 6379
      password: Chinaunicom@1358
      database: 0
      timeout: 10s
YEOF
```

### 6.4 启动服务

```bash
# 前台启动（调试用）
java -jar /opt/ruoyi-admin.jar \
  --spring.config.additional-location=/opt/ruoyi/application-druid.yml \
  --snmp.walk-path=snmpwalk

# 后台启动（生产用）
nohup java -jar /opt/ruoyi-admin.jar \
  --spring.config.additional-location=/opt/ruoyi/application-druid.yml \
  --snmp.walk-path=snmpwalk \
  > /opt/ruoyi/app.log 2>&1 &

# 验证
curl http://localhost:8080/
# 返回: 欢迎使用智慧园区管理系统后台管理框架，当前版本：v1.0.0
```

### 6.5 创建 systemd 服务（推荐）

```bash
cat > /etc/systemd/system/ruoyi.service << 'EOF'
[Unit]
Description=智慧园区管理系统
After=network.target mysql.service redis-server.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/ruoyi
ExecStart=/usr/bin/java -jar /opt/ruoyi-admin.jar \
  --spring.config.additional-location=/opt/ruoyi/application-druid.yml \
  --snmp.walk-path=snmpwalk
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable ruoyi
systemctl start ruoyi

# 管理命令
systemctl status ruoyi   # 查看状态
systemctl restart ruoyi  # 重启
systemctl stop ruoyi     # 停止
journalctl -u ruoyi -f   # 查看日志
```

### 6.6 JVM 调优参数

```bash
# 生产环境建议添加以下参数：
java -Xms1024m -Xmx2048m \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -jar /opt/ruoyi-admin.jar ...
```

## 7. 前端部署（Nginx）

### 7.1 上传前端文件

将本地构建好的 `ruoyi-ui/dist/` 目录上传：

```bash
scp -r ruoyi-ui/dist/* root@192.168.1.60:/opt/ruoyi/html/

# 设置权限
ssh root@192.168.1.60 "chown -R www-data:www-data /opt/ruoyi/html/"
```

### 7.2 配置 Nginx

```bash
cat > /etc/nginx/sites-available/ruoyi << 'NEOF'
server {
    listen       80;
    server_name  192.168.1.60;    # 修改为实际域名/IP

    root   /opt/ruoyi/html;
    index  index.html;

    # API代理到后端 (生产环境路径)
    location /prod-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 开发环境API代理（前端npm run dev时使用）
    location /dev-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket代理（视频流、考勤实时推送）
    location /ws/ {
        proxy_pass http://127.0.0.1:8080/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 3600s;
    }

    # 文件上传路径
    location /profile/ {
        proxy_pass http://127.0.0.1:8080/profile/;
    }

    # 静态资源（HTML5 History模式）
    location / {
        try_files $uri $uri/ /index.html;
        expires 7d;
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
}
NEOF

# 启用站点
ln -sf /etc/nginx/sites-available/ruoyi /etc/nginx/sites-enabled/ruoyi
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重载
systemctl reload nginx
```

## 9. 钉钉/企业微信对接配置

### 9.1 考勤回调配置

进入系统菜单：**考勤管理 → 数据对接 → 应用管理**

#### 钉钉配置

1. 登录 [钉钉开放平台](https://open.dingtalk.com)
2. 创建企业内部应用 → 获取 AppKey / AppSecret
3. 进入"事件订阅" → 配置回调URL
4. 在本系统"应用管理"中创建钉钉应用，自动生成 Token/AESKey/回调URL
5. 回调地址示例：`http://你的域名/attendance/callback/dingtalk`
6. 订阅事件：`attendance_check_record`、`attendance_leave_record`、`attendance_overtime_record`

#### 企业微信配置

1. 登录 [企业微信管理后台](https://work.weixin.qq.com)
2. 应用管理 → 自建应用 → 获取 CorpID / Secret
3. 接收消息 → 设置API接收
4. 在本系统"应用管理"中创建企业微信应用
5. 回调地址示例：`http://你的域名/attendance/callback/wecom`

### 9.2 组织架构同步

进入：**人员管理 → 组织同步**

两种同步方式：

| 方式 | 说明 |
|------|------|
| **API一键同步** | 填写 AppKey/Secret 后自动调用钉钉/企微API拉取 |
| **JSON手动导入** | 粘贴从钉钉/企微导出的组织架构JSON数据 |

### 9.3 安全配置（重要）

回调URL需加入 Spring Security 白名单。已在 `SecurityConfig.java` 中配置：

```java
"/attendance/callback/**",  // 考勤回调
"/ws/**"                     // WebSocket
```

若修改了回调路径，需同步更新此配置。

### 11.3 备份策略

#### 数据库备份

```bash
# 每日全量备份脚本
cat > /opt/ruoyi/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/ruoyi/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
mysqldump -u sadmin -p'Chinaunicom@1358' --single-transaction ry101 \
  | gzip > $BACKUP_DIR/ry101_${DATE}.sql.gz
# 保留最近7天
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete
echo "Backup completed: ry101_${DATE}.sql.gz"
EOF

chmod +x /opt/ruoyi/backup.sh

# 添加到crontab（每天凌晨3点）
# crontab -e
# 0 3 * * * /opt/ruoyi/backup.sh >> /opt/ruoyi/backup.log 2>&1
```

#### 应用备份

```bash

  /opt/ruoyi-admin.jar \
```

### 11.6 常见问题

| 问题 | 排查方法 |
|------|------|
| 后端启动失败 | `tail -100 /opt/ruoyi/app.log` 查看错误 |
| 登录返回500 | 检查MySQL连接、验证码Redis状态 |
| Nginx 502 | 确认后端8080端口正常、`systemctl status ruoyi` |
| 前端空白页 | 检查 `/opt/ruoyi/html/index.html` 存在、Nginx配置正确 |
| SNMP不可用 | `which snmpwalk` 确认已安装、`snmpwalk --version` |
| WebSocket断开 | 检查Nginx `/ws/` 代理配置 |
